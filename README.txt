一、输入说明
1、两类合法的请求格式分别为
	(1)楼层请求：(F_R,floor,Up/Down)
	(2)电梯请求：(E_R,floor)
	两类请求格式中，F_R/E_R为标识符，表示当前请求类型，floor为整型楼层变量(目前支持[1,6]的整数),表示当前请求目的楼层，Up/Down为方向变量，表示楼层请求的方向。
2、请求序列
	每一个请求（包括最后一个请求）结束后允许插入逗号(不插入也可)作为分隔符，请求序列以回车结束。
3、补充说明
	允许在任意一个请求开始前、任意一个请求结束后和任意一个请求内部的标识符、变量、括号、逗号之间插入任意数量的空格。
	所有英文字母大小写敏感，逗号、括号为半角标点。
	楼层请求中，1楼只允许发送Up请求，6楼只允许发送Down请求

4、合法输入序列示例：
(F_R,5,Up),(E_R,1)(F_R,6,Down)(E_R,3)(F_R,2,Up)(E_R,6)

二、输出说明
1、输出格式说明
	在请求序列合法的情况下，将会输出电梯运行过程中的途径楼层、运行方向和运行状态。每处理完一个请求，将输出电梯在处理本次请求过程中除初始状态外的所有状态，每一个状态输出一行。输出格式为(floor,Up/Down,Pass/Stop)
	其中，floor为楼层变量，表示电梯当前所在楼层，Up/Down为方向变量，表示电梯运行方向，Pass/Stop为状态变量，表示当前电梯是停靠该楼层还是通过不停靠该楼层。
	若请求序列不合法，将会输出“Illegal Request”
2、电梯运行说明
	电梯的初始状态为：停靠在1楼，运行方向向上。
	电梯运行到1楼后，运行方向将自动调整为向上；电梯运行到6楼后，运行方向将自动调整为向上。
5、输出示例
	对应于上方合法输入示例的输出为：
	(2,Up,Pass)
	(3,Up,Pass)
	(4,Up,Pass)
	(5,Up,Stop)
	(4,Down,Pass)
	(3,Down,Pass)
	(2,Down,Pass)
	(1,Up,Stop)
	(2,Up,Pass)
	(3,Up,Pass)
	(4,Up,Pass)
	(5,Up,Pass)
	(6,Down,Stop)
	(5,Down,Pass)
	(4,Down,Pass)
	(3,Down,Stop)
	(2,Down,Stop)
	(3,Up,Pass)
	(4,Up,Pass)
	(5,Up,Pass)
	(6,Down,Stop)
